# 历史配置对比

历史配置对比

221107：

新增 新增 dailyBattery 每日电池任务，发送 5 条弹幕获取 1 电池。
新增 新增多用户配置支持公共配置。
修复 【非云函数】，cookie 存在单引号时重新保存错误。
修复 【云函数】，阿里云内不打印错误日志。
优化 增加风纪任务的可配置项，参考观点时默认屏蔽 [放弃投票]。
优化 清除默认配置中过期的活动链接。
优化 【非云函数】，默认日志也会清空 15 日前的记录。
优化 漫画每日阅读无需额外配置，开启即可使用。
优化 优化自定义推送 API 的替换逻辑。

# 0.6.221014

修复 【非云函数】，转盘抽奖错误的判断为今日已运行。
新增 为亲密度增加运行后检查运行情况，默认不开启，因为 b 站数据延迟可能会出现检查后从头运行。
新增 【仅云函数】，风纪委员在没有新案件需要休眠时，采用新建 Trigger 等待的方式（通过调用次数换取资源时间）。默认开启 [jury.newTrigger]。
新增 【非云函数】，避免出现某些人所说的莫名奇妙的 cookie 失效（当然只是尝试，本人并遇到过），配置 createCookieDay，运行通过当前 cookie 创建出新的 cookie，（非刷新，cookie 可以共存）。
优化 为亲密度减少某些情况下的 api 调用。
新增 漫画每日首次分享。
新增 【合作视频】精准匹配用户，视频上传者必须是目标用户。这是曾经有过的功能，后来因为逻辑的变动和 api 的更改就废弃了。
优化 获取指定类型稿件的方式由成功获取后进行判断，修改为只查找指定类型。
优化 稿件来源可自定优先顺序和去掉来源。
优化 多账号中错误的配置不再加载时直接跳过，改为运行时跳过。

# 0.6.220930

优化 增加消息推送仅错误时推送。
优化 未登录时打印 id，方便为多账号检查。
优化 删除风纪委员的 mode 配置，仅支持原 mode=1。
修复 更新依赖。
修复 漫画领取奖励出现赛季错误的问题。
修复 充电/兑换电池若干问题。
优化 去除投币功能部分废弃的配置。
优化 转盘抽奖的在线链接支持自定义/代理前缀。
优化 调整部分日志，增加错误标题。
优化 增加取关的配置。
优化 增加赠送直播间礼物 all 选项。
删除 去除自动更新。

# 0.5.220823

新增 取消 charging，使用 useCouponBp 替代，并增加 b 币兑换电池（群友需求）。
新增 mangaTask 增加漫画自动领取奖励（阅读奖励，签到奖励，不过前提是这些已经做过了），暂时不需要开启，也不能关闭。
新增 转盘的爬虫增加追番选项，如有需要可以配置 [activityLottery.bangumi]，当然是否有用存在随机性。
新增 addCoins 投币不再是固定 1 颗。
新增 bigPoint 重试可配置等待时间。
新增 npm 增加 --createCookie -cck 参数，可以使用 cookie 生成新 cookie，用处懂的都懂。
修复 某些情况下替换 cookie 导致配置文件格式错误（群友反馈）。
优化 完善 delay 参数。
优化 调整 matchGame 默认值（更加保守），防止有人不懂直接开启但不改配置，导致丢失大量硬币（冤大头群友）。
优化 部分情况的日志输出优化，增加中奖标题提示（例如：xxxxx-【转盘】）（Issue 92）。
优化 getPrivilege 的日志增加 73319 的解释（Issue 109）。
优化 优化输出的版本。
优化 tsconfig strictNullChecks 仅改变开发体验，增加稳定性，与程序最后运行结果无关。

  • 优化 bigPoint 增加重试、取消观看视频和自定义视频（正片，非 up 投稿）的配置。

liveIntimacy 的直播心跳可能需要消耗比实际需要时间长很多的时间，这是因为目前对刚获取到的亲密度判断不准确，甚至已经发送弹幕和点赞了，但是依然会通过直播心跳去获取这多出来的 200 亲密度，浪费 10 分钟，（虽然这部分是异步操作的，你的粉丝牌越多，浪费的时间越接近于 0）。
