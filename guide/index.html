# 支持功能

注意：

  • 表中排列顺序即为功能的调用顺序，内部设置顺序，与配置顺序无关。
  • 带删除线的表示已经弃用或其它原因不推荐使用，但是内容没有删除。
功能默认值说明配置文档
exchangeCoupon兑换漫读券 （暂时只能通过指定 task 运行）点击
liveSignTasktrue直播间签到
addCoinstrue投币点击
bigPoint大会员领取大积分点击
shareAndWatchtrue观看和分享视频
silver2Cointrue银瓜子兑换硬币（配合直播签到用）
mangaTask漫画任务（包含签到，分享，阅读，小游戏）点击
supGroupSign应援团签到 （目前也无意义了啊）
useCouponBp使用 b 币券（充电/兑换电池）点击
getVipPrivilege获取大会员权益
matchGame赛事竞猜（人人都是硬币慈善家）点击
giveGift赠送过期礼物点击
liveIntimacy牌子亲密度 （非云函数为异步运行）点击
batchUnfollow取消关注点击
liveLottery直播天选时刻（黑号风险）点击
liveRedPack直播天选红包点击
dailyBattery每日电池任务（电池+1）
activityLottery转盘抽奖点击
judgement风纪委员点击

只有将这些值设置为 true 才会生效，如果设置为 false ，具体的配置再怎么样也不会去执行，下面是参考写法。

参考写法
{
  function: {
    silver2Coin: true,
    addCoins: true,
    liveSignTask: true,
    shareAndWatch: true,
    mangaTask: false,
    batchUnfollow: false,
    supGroupSign: false,
    useCouponBp: false,
    getVipPrivilege: false,
    giveGift: false,
    matchGame: false,
    liveRedPack: false,
    supGroupSign: false,
    bigPoint: false,
    judgement: false,
    activityLottery: false,
  },
}

# 更新

测试内容

漫画基本上获取积分的方法都有了，兑换也有了，使用也有。相当于来了个更高级的等免功能

历史各版本配置对比

下次更新预览（预计 2023 年或者农历 2023）：

警告 删除部分旧配置兼容。
警告 不再提供打包可执行文件。
新增 通过扫码获取 Cookie，二维码可打印到青龙日志，云函数控制台，本地控制台。
新增 Docker 可以传递参数。
修复 漫画阅读额外使用了非即将过期的漫读券。
修复 大积分任务未全部完成的情况下没有日志输出。

在做但没有思路的：

修改 json5 文件。操作 json5 和 json 不一样，因为 json5 有注释，目前 json5 官方提供的 js 包并不支持修改后保留注释。comment-json 倒是有这种功能，但奈何不是 json5。从一开始就选错了？可能只有在 json5 官方库的基础上自己写一个了。

上一个版本 1203：

警告 删除大部分旧配置兼容。
警告 删除 serverless 部署工具。
新增 [manga.guess] 漫画每日猜拳（间隔太小会出现未知问题，故运行时长较长。收益存在随机性，也可能赔本，但基本上不存在赔本）。
新增 每月会员福利会领取 [漫画商城优惠]。
新增 多账户下风纪委员长期没案件会阻塞后期运行，现可以配置为异步执行。
新增 漫画券兑换配置更改，增加保留积分数、兑换数在原基础上增加自动数量。
修复 修复 dailyBattery 任务只部分完成的问题。
修复 粉丝亲密度 intimacy 错误的结束、错误的重试机制。
优化优化 大积分每日 75 调整到 65（官方删除一个任务）。
优化 大积分视频观看：去掉自定义 epid（如有自定义需要可以提，没人需要就不加了）、默认西游记随机一集改为四大名著 5 个正剧中随机、完善接口数据，更加还原真实请求。
优化 漫画券兑换未到达 12 点时进行等待，超过 12:02 自动退出，故可提前启动。
优化 漫画阅读快速完成。
优化 当然还有每次更新的保留节目：日志调整。
